## How it grows

A GitHub Action runs every night and:

1. Pulls the last 7 days of commit counts via `git log --since`
2. Maps each day's count to a building height on a log-ish scale, so one huge refactor day doesn't flatten the rest of the week into nothing
3. Rewrites `assets/skyline.svg` with new bar heights and window positions
4. Commits the new SVG back to the repo

The skyline above is from *this* week, not a screenshot frozen in time. Come back Monday and it's a different city.

No charting library, no API, no LLM — just `git log`, a small script, and raw SVG templating.

<summary>⚙️ The actual generator script</summary>

```python
import subprocess, datetime

def commits_per_day():
    counts = []
    for i in range(7):
        day = datetime.date.today() - datetime.timedelta(days=i)
        since = day.strftime("%Y-%m-%d 00:00")
        until = day.strftime("%Y-%m-%d 23:59")
        out = subprocess.run(
            ["git", "log", f"--since={since}", f"--until={until}", "--oneline"],
            capture_output=True, text=True
        )
        counts.append(len(out.stdout.strip().splitlines()))
    return list(reversed(counts))

def build_svg(counts):
    max_h = 180
    bars = ""
    for i, c in enumerate(counts):
        h = min(max_h, 20 + c * 25)
        x = 40 + i * 80
        y = 200 - h
        bars += f'<rect x="{x}" y="{y}" width="60" height="{h}" fill="#415a77"/>'
    return f'''<svg width="640" height="240" xmlns="http://www.w3.org/2000/svg">
        <rect width="640" height="200" fill="#0d1b2a"/>
        {bars}
    </svg>'''

if __name__ == "__main__":
    svg = build_svg(commits_per_day())
    with open("assets/skyline.svg", "w") as f:
        f.write(svg)
```

```yaml
# sources/.github/workflows/skyline.yml
name: skyline
on:
  schedule:
    - cron: '0 4 * * *'
jobs:
  redraw:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with: { fetch-depth: 0 }
      - run: python3 sources/scripts/build_skyline.py
      - run: |
          git config user.name "skyline-bot"
          git config user.email "actions@users.noreply.github.com"
          git add assets/skyline.svg
          git commit -m "chore: redraw skyline" || echo "no change"
          git push
```

## Bonus layer: a city that thinks

A second, smaller cellular automaton grid is seeded by hashing the day's first commit SHA into a starting bit pattern, then run forward as Conway's Game of Life. Same mechanism, same repo — but the seed is genuinely derived from real activity instead of `random.seed()`, so it's deterministic and reproducible from the commit log alone.

```python
import hashlib

def seed_from_commit(sha: str, grid_size: int = 16):
    digest = hashlib.sha256(sha.encode()).hexdigest()
    bits = bin(int(digest, 16))[2:].zfill(256)
    cells = [int(b) for b in bits[:grid_size * grid_size]]
    return [cells[i:i+grid_size] for i in range(0, len(cells), grid_size)]
```
